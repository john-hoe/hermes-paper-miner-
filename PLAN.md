# Paper Miner - 执行方案 (V1)

## 1. 核心数据结构与边界定义 (TDD)
- **成功用例**: 每日早 8:00，成功抓取 HF Daily Papers，若存在 ≥85 分的论文，推送如下格式：
```markdown
📄 [92分] [Meta AI] Voicebox-V2：零样本极速声音克隆
🔗 [论文链接] | [代码库: ✅已开源]
💡 这是啥：提出了一种新的流式匹配算法...
🛠 落地场景预测（你可以怎么用）：
 1. 出海短视频配音：批量生成外语配音。
 2. AI NPC 游戏客服：零延迟拟真客服。
⚠️ 落地劝退点：单卡 24GB 显存起步，边缘设备不可用。
```
- **边缘用例1 (无高分)**: 推送 `🚫 今日无高分好文，最高分为75分，已跳过推送`
- **边缘用例2 (系统异常)**: 捕获异常，仅推送 `⚠️ 系统故障：[错误简述]，今日淘金任务失败`。绝不抛出长堆栈。

## 2. 架构设计与数据流
1. **Fetch**: `requests` 获取 `https://huggingface.co/api/daily_papers` (Top 5)。
2. **Score & Brainstorm**: 
   - 将标题+摘要组装进特定 Prompt。
   - 调用你本地/指定的大模型接口 (如 OpenAI 格式的本地服务)。
   - 强制模型输出 JSON 格式（得分、总结、场景、劝退点、机构信息）。
3. **Format**: 将 JSON 解析为 Markdown 字符串。
4. **Push**: 使用 Telegram Bot API 发送。

## 3. 打分权重分配
- 满分 100分 + 附加分
- **工程落地性 (40%)**: 是否有代码？复现门槛？
- **痛点解决度 (30%)**: 解决真实痛点 vs 跑分刷榜。
- **创新性 (20%)**: 思路新颖度。
- **出圈潜力 (10%)**: 商业产品化可行性。
- **附加分 (+5~10分)**: 大厂 (Meta, Google, OpenAI等) 或 顶尖高校。
