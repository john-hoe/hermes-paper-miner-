# Paper Miner - 执行方案 (V1)

## 1. 核心数据结构与边界定义 (TDD)
- **成功用例**: 每日早 8:00，成功抓取 HF Daily Papers，过滤掉已推送过的论文。若存在 ≥85 分的论文，推送如下格式：
```markdown
📄 [92分] [Meta AI] Voicebox-V2：零样本极速声音克隆
🔗 [论文链接] | [代码库: ✅已开源]
🎯 命中偏好：[多模态出片/Agent协作/本地LLM]
💡 这是啥：提出了一种新的流式匹配算法...
🛠 落地场景预测（你可以怎么用）：
 1. 出海短视频配音：批量生成外语配音。
 2. AI NPC 游戏客服：零延迟拟真客服。
⚠️ 落地劝退点：单卡 24GB 显存起步，边缘设备不可用。
```
- **边缘用例1 (无高分)**: 推送 `🚫 今日无高分好文，最高分为75分，已跳过推送`
- **边缘用例2 (系统异常)**: 捕获异常，仅推送 `⚠️ 系统故障：[错误简述]，今日淘金任务失败`。绝不抛出长堆栈。

## 2. 架构设计与核心机制
1. **Fetch (去重机制)**: 
   - 抓取 `https://huggingface.co/api/daily_papers`。
   - 读取本地 `seen_papers.json`（轻量级去重库），过滤掉历史已经处理过的论文 ID。
2. **Dynamic Preferences (动态偏好注入)**:
   - 读取外置配置文件 `user_preferences.yaml` 或 `json`。包含当前的关注点（如：Mac本地大模型跑满、多Agent协作出片、QA自动化测试等）和排斥点（纯生物医药、纯理论公式推导等）。
   - 若命中偏好，直接给予加分，并在推送时标记 `🎯 命中偏好`。
   - **偏好保鲜机制**：每隔 14 天（或月末），在当天的推送末尾自动追加一行提醒：`🔔 提示：距离上次更新您的私人AI偏好已过14天，如有变动，请随时告诉我更新。`
3. **Score & Brainstorm**: 
   - 组装 Prompt（带上动态偏好），调用大模型接口，强制输出 JSON。
4. **Format & Push**: 
   - Markdown 格式化并发送 Telegram。最后将成功推送的论文 ID 写入 `seen_papers.json` 永久封存。
